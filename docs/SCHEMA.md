

## New Tables (Build These)

### `water_fear_logs`
Daily water fear level — critical path metric for Aquaman checkpoint.

```sql
CREATE TABLE water_fear_logs (
    id              SERIAL PRIMARY KEY,
    date            TEXT NOT NULL,
    fear_level      INTEGER NOT NULL CHECK (fear_level BETWEEN 1 AND 10),
    context_note    TEXT,           -- Optional free text
    session_type    TEXT,           -- 'pool', 'open_water', 'general'
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### `ironman_training_plan`
The source-of-truth triathlon periodization plan. Generated by training_plan.py.

```sql
CREATE TABLE ironman_training_plan (
    id              SERIAL PRIMARY KEY,
    week_number     INTEGER NOT NULL,       -- Week 1 = first week of plan
    phase           TEXT NOT NULL,          -- 'base', 'build', 'race_prep', 'taper'
    date            DATE NOT NULL,
    day_of_week     INTEGER NOT NULL,       -- 0=Mon, 6=Sun
    discipline      TEXT NOT NULL,          -- 'swim', 'bike', 'run', 'brick', 'rest'
    session_name    TEXT NOT NULL,
    duration_mins   INTEGER,
    hr_target_low   INTEGER,                -- Zone 2 lower bound
    hr_target_high  INTEGER,                -- Zone 2 upper bound
    distance_target FLOAT,                  -- km
    description     TEXT,
    notes           TEXT,
    is_brother_session BOOLEAN DEFAULT FALSE,
    garmin_scheduled BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### `principle_compliance`
Daily principle compliance scores — feeds probability calculation.

```sql
CREATE TABLE principle_compliance (
    date                    TEXT PRIMARY KEY,
    zone2_compliant         BOOLEAN,        -- Was session in Zone 2?
    zone2_avg_hr            INTEGER,        -- Actual avg HR
    life_load_score         INTEGER,        -- Workday load 1-10
    water_fear_level        INTEGER,        -- 1-10 (from water_fear_logs)
    session_completed       BOOLEAN,        -- Did session happen?
    session_type            TEXT,           -- discipline
    principle_04_aligned    BOOLEAN,        -- Was it a brother session if applicable?
    notes                   TEXT,
    created_at              TIMESTAMPTZ DEFAULT NOW()
);
```

### `probability_snapshots`
Weekly probability score history for trend visualization.

```sql
CREATE TABLE probability_snapshots (
    id                      SERIAL PRIMARY KEY,
    date                    DATE NOT NULL,
    overall_score           INTEGER,        -- 0-100
    zone2_component         INTEGER,        -- 0-100
    consistency_component   INTEGER,        -- 0-100
    life_load_component     INTEGER,        -- 0-100
    swim_frequency_component INTEGER,       -- 0-100
    notes                   TEXT,
    created_at              TIMESTAMPTZ DEFAULT NOW()
);
```

## FastAPI Endpoints (api/main.py)

```
GET  /api/today          → today's metrics, readiness, calories, plan
GET  /api/probability    → current score + breakdown + 30-day trend
GET  /api/week           → last 7 days: sessions, compliance, fear
GET  /api/plan           → next 14 days from ironman_training_plan
GET  /api/stats          → baseline fitness stats (VO2, FTP, swim pace)
GET  /api/checkpoints    → days until each event + readiness per event
GET  /health             → health check for Vercel
```

All endpoints return JSON. No authentication required (single-user system,
no sensitive data exposed beyond fitness metrics).
